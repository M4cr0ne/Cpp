BUBBLE SORT

Il Bubble Sort è un algoritmo di ordinamento per confronto che lavora esaminando coppie di elementi adiacenti. Quando due valori consecutivi risultano nell'ordine sbagliato, viene eseguito uno scambio. Ripetendo questo procedimento, i valori più grandi vengono progressivamente spinti verso la parte finale del vettore.

1. Idea chiave

Il Bubble Sort non cerca direttamente il minimo o il massimo in tutto il vettore. Lavora invece su confronti locali: si osservano due elementi vicini e, se risultano invertiti, vengono scambiati.

Al termine di ogni passaggio completo, almeno un elemento raggiunge la propria posizione definitiva: nel caso di ordinamento crescente, il più grande tra quelli ancora non sistemati finisce in fondo.

In sostanza: l'algoritmo non costruisce l'ordine “scegliendo” subito il valore corretto, ma corregge progressivamente le inversioni presenti tra elementi adiacenti.

2. Problema che risolve

L'obiettivo è ordinare una sequenza di valori, ad esempio numeri interi, dal più piccolo al più grande.

Input

Un vettore non ordinato, ad esempio {29, 10, 14, 37, 13}.

Output

Lo stesso vettore ordinato in modo crescente: {10, 13, 14, 29, 37}.

È particolarmente utile in fase iniziale perché rende molto visibili il confronto, la condizione di scambio e il significato dell'ordine crescente.

3. Modello mentale

Si può immaginare che gli elementi più grandi “risalgano” verso destra ad ogni passaggio, come bolle che salgono verso la superficie. Da qui deriva il nome dell'algoritmo.

4. Procedimento passo-passo

Si osservi il comportamento dell'algoritmo sul vettore {29, 10, 14, 37, 13}.

Passaggio 1

Si confrontano 29 e 10: essendo nell'ordine sbagliato, si scambiano. Poi si confrontano 29 e 14: nuovo scambio. Il confronto tra 29 e 37 non richiede interventi. Infine si confrontano 37 e 13: si scambiano.

Risultato: {10, 14, 29, 13, 37}

A questo punto 37 è sicuramente in posizione definitiva.

Passaggio 2

Si riparte dall'inizio, ma non serve più arrivare fino all'ultimo elemento già sistemato. I confronti portano allo scambio tra 29 e 13.

Risultato: {10, 14, 13, 29, 37}

Ora anche 29 è in posizione definitiva.

Passaggio 3

Si confrontano le coppie ancora utili e si esegue lo scambio tra 14 e 13.

Risultato: {10, 13, 14, 29, 37}

Passaggio 4

Se si usa la versione più semplice, il passaggio avviene comunque, anche se non ci sono scambi. Se invece si usa una variabile booleana che registra la presenza di scambi, l'algoritmo può terminare in anticipo appena il vettore risulta già ordinato.

Ogni confronto riguarda elementi adiacenti Ad ogni passaggio si fissa un elemento in fondo L'ottimizzazione con swapped consente uscita anticipata
Alla fine dell'iterazione esterna di indice i, la porzione finale del vettore, lunga i + 1, risulta già ordinata e contiene gli elementi più grandi nelle loro posizioni definitive.

    fine per

    se swapped è falso

    fine se
fine per
La variabile swapped segnala se durante il passaggio è avvenuto almeno uno scambio. Se non si verifica alcuno scambio, il vettore risulta già ordinato e non serve proseguire.

6. Codice C++ commentato

Di seguito è riportata una versione chiara e didattica dell'algoritmo, applicata a un vector<int>.

#include <iostream>
#include <vector>
using namespace std;

int main()
{
    vector<int> numbers = {29, 10, 14, 37, 13};
    int n = numbers.size();

    // Outer loop: each pass moves the largest remaining value to the end
    for (int i = 0; i < n - 1; i++)
    {
        bool swapped = false;

        // Inner loop: compares adjacent elements in the unsorted part
        for (int j = 0; j < n - 1 - i; j++)
        {
            if (numbers[j] > numbers[j + 1])
            {
                int temp = numbers[j];
                numbers[j] = numbers[j + 1];
                numbers[j + 1] = temp;
                swapped = true;
            }
        }

        // If no swap happened, the vector is already sorted
        if (!swapped)
        {
            break;
        }
    }

    cout << "Sorted vector: ";
    for (int value : numbers)
    {
        cout << value << " ";
    }

    return 0;
}
Ciclo esterno

Conta i passaggi completi e riduce progressivamente la parte del vettore ancora da controllare.

Ciclo interno

Confronta coppie consecutive e decide se eseguire oppure no lo scambio.

Lettura delle variabili principali

Variabile Ruolo Osservazione didattica
i Conta il numero di passaggi effettuati. Dopo ogni passaggio cresce la parte finale già sistemata del vettore.
j Scorre la parte ancora non ordinata del vettore. Serve per confrontare l'elemento in indice j con quello in indice j + 1.
swapped Registra se nel passaggio corrente è avvenuto almeno uno scambio. Permette di interrompere l'algoritmo quando il vettore è già ordinato.
temp Variabile di appoggio per lo scambio. Evita la perdita del valore contenuto in numbers[j].

7. Complessità e prestazioni

Aspetto Valore Significato
Tempo nel caso migliore O(n) Con la variabile swapped, un vettore già ordinato richiede un solo passaggio senza scambi.
Tempo nel caso medio O(n²) Il numero complessivo di confronti cresce quadraticamente.
Tempo nel caso peggiore O(n²) Quando molti elementi sono lontani dalla posizione corretta, servono molti passaggi e molti scambi.
Numero massimo di confronti n(n-1)/2 Dipende dalla somma dei confronti eseguiti nei passaggi successivi.
Numero di scambi Può essere elevato Ogni inversione locale viene corretta tramite uno scambio immediato.
Spazio extra O(1) Usa solo poche variabili aggiuntive.
Stabilità Sì Se si scambia solo quando a[j] > a[j+1], l'ordine relativo di elementi uguali viene preservato.
Adattività Parziale Con l'ottimizzazione swapped può fermarsi prima se il vettore è già ordinato o quasi ordinato.
Il costo quadratico deriva dal fatto che, nel caso generale, il ciclo interno esegue circa n-1 confronti nel primo passaggio, n-2 nel secondo, e così via. La somma (n-1) + (n-2) + ... + 1 vale n(n-1)/2.
Il Bubble Sort è utile per comprendere il meccanismo degli scambi locali, ma in applicazioni reali non rappresenta una scelta efficiente per insiemi di dati estesi.

8. Confronto con altri algoritmi

Conviene confrontarlo con altri algoritmi elementari per cogliere differenze reali di comportamento, non solo di risultato finale.

Algoritmo Idea principale Punto forte Limite principale
Bubble Sort Corregge inversioni tra elementi adiacenti Molto intuitivo dal punto di vista visivo Richiede molti scambi
Selection Sort Cerca il minimo e lo porta davanti Effettua pochi scambi Non sfrutta il fatto che il vettore sia già quasi ordinato
Insertion Sort Inserisce ogni elemento nella posizione corretta della parte già ordinata Molto efficace su dati piccoli o quasi ordinati Resta quadratico nel caso peggiore

Quando è utile studiarlo

Quando si vuole far comprendere in modo diretto il ruolo del confronto, della condizione e dello scambio tra due elementi adiacenti.

Quale confronto didattico è più utile

Confrontarlo con il Selection Sort è molto istruttivo: entrambi sono quadratici, ma il Bubble Sort scambia spesso, mentre il Selection Sort seleziona prima e scambia dopo.

9. Errori tipici

Errore 1 • Confondere gli indici del confronto

Nel Bubble Sort si confrontano sempre j e j + 1. Non si confronta l'elemento corrente con un minimo o con un massimo globale.

Errore 2 • Sbagliare il limite del ciclo interno

Il limite corretto dipende dal numero di passaggi già completati. Scrivere j < n - 1 - i evita di ricontrollare inutilmente la parte finale già ordinata.

Errore 3 • Dimenticare di reimpostare swapped a false

La variabile deve essere azzerata all'inizio di ogni passaggio esterno. In caso contrario, l'uscita anticipata non funziona correttamente.

Errore 4 • Pensare che ogni passaggio ordini tutto il vettore

Ogni passaggio garantisce solo che il massimo della parte analizzata finisca in fondo. L'ordinamento completo richiede più iterazioni.

12. Riepilogo

  • Il Bubble Sort è un algoritmo di ordinamento per confronto.
  • Confronta elementi adiacenti ed esegue uno scambio quando sono nell'ordine sbagliato.
  • Al termine di ogni passaggio il valore massimo della parte analizzata si trova in fondo.
  • Lavora in-place e usa pochissima memoria aggiuntiva.
  • Con l'ottimizzazione swapped può terminare prima in caso di vettore già ordinato.
  • Nel caso medio e peggiore ha complessità temporale O(n²).
Quando il comportamento del Bubble Sort risulta chiaro, diventa più semplice comprendere il significato di stabilità, adattività e costo degli scambi, e quindi confrontare in modo più maturo ordinamenti diversi.
