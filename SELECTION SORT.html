SELECTION SORT

Il Selection Sort è un algoritmo di ordinamento per confronto: ad ogni passaggio individua il valore minimo nella parte ancora disordinata del vettore e lo colloca nella prima posizione utile. È particolarmente utile per comprendere la logica degli ordinamenti, perché separa in modo molto chiaro la parte già sistemata da quella ancora da analizzare.

1. Idea chiave

Il Selection Sort divide idealmente il vettore in due parti: una parte iniziale già ordinata e una parte finale ancora da ordinare.

Ad ogni passaggio cerca il valore più piccolo nella parte non ordinata e lo scambia con il primo elemento di quella porzione.

In sostanza: non sposta subito ogni valore al posto giusto, ma decide ogni volta quale deve essere il prossimo elemento corretto da mettere in testa alla parte ancora disordinata.

2. Problema che risolve

Questo algoritmo serve a ordinare una sequenza di valori, per esempio numeri interi, dal più piccolo al più grande.

Input

Un vettore non ordinato, ad esempio {29, 10, 14, 37, 13}.

Output

Lo stesso vettore ordinato in modo crescente: {10, 13, 14, 29, 37}.

È uno degli algoritmi più adatti per iniziare lo studio degli ordinamenti, perché la logica è lineare e ogni passaggio è facilmente osservabile.

3. Modello mentale

Si può rappresentare il vettore come una fila di caselle. All'inizio nessuna posizione è ancora garantita come corretta. Il Selection Sort stabilisce, un passaggio dopo l'altro, quale valore deve occupare la prossima posizione definitiva.

4. Procedimento passo-passo

Si osservi il comportamento dell'algoritmo sul vettore {29, 10, 14, 37, 13}.

Passaggio 1

Si cerca il minimo nell'intero vettore. Il minimo è 10, che si trova in indice 1. Si scambia con il valore in indice 0.

Risultato: {10, 29, 14, 37, 13}

Passaggio 2

Ora l'indice 0 è sistemato. Si cerca il minimo da indice 1 fino alla fine. Il minimo è 13. Si scambia con 29.

Risultato: {10, 13, 14, 37, 29}

Passaggio 3

Si cerca il minimo da indice 2 alla fine. Il minimo è già 14, quindi non serve alcuno scambio.

Risultato: {10, 13, 14, 37, 29}

Passaggio 4

Si cerca il minimo da indice 3 alla fine. Il minimo è 29, quindi si scambia con 37.

Risultato finale: {10, 13, 14, 29, 37}

Ogni passaggio fissa un elemento corretto Il numero di passaggi principali è n-1 Il minimo viene cercato con un ciclo interno
Alla fine di ogni iterazione del ciclo esterno vale una proprietà importante: la porzione da indice 0 a indice i risulta già ordinata e contiene esattamente i valori più piccoli del vettore.

5. Pseudocodice

Questo pseudocodice mostra la logica dell'algoritmo senza concentrarsi ancora sulla sintassi precisa del C++.

La variabile minIndex non contiene il valore minimo, ma l'indice in cui quel valore si trova.

6. Codice C++ commentato

Di seguito è riportata una versione chiara e didattica dell'algoritmo, applicata a un vector<int>.

#include <iostream>
#include <vector>
using namespace std;

int main()
{
    vector<int> numbers = {29, 10, 14, 37, 13};
    int n = numbers.size();

    // Outer loop: selects the position that must receive the next minimum value
    for (int i = 0; i < n - 1; i++)
    {
        int minIndex = i;

        // Inner loop: searches for the minimum value in the unsorted part
        for (int j = i + 1; j < n; j++)
        {
            if (numbers[j] < numbers[minIndex])
            {
                minIndex = j;
            }
        }

        // Swap only if a smaller value was found
        if (minIndex != i)
        {
            int temp = numbers[i];
            numbers[i] = numbers[minIndex];
            numbers[minIndex] = temp;
        }
    }

    cout << "Sorted vector: ";
    for (int value : numbers)
    {
        cout << value << " ";
    }

    return 0;
}
Ciclo esterno

Stabilisce quale posizione del vettore deve essere sistemata in quel momento.

Ciclo interno

Esamina tutti gli elementi successivi per trovare l'indice del valore più piccolo.

Lettura delle variabili principali

Variabile Ruolo Osservazione didattica
i Indica la posizione che deve ricevere il prossimo valore corretto. La parte a sinistra di i è già considerata sistemata.
j Scorre la parte non ordinata del vettore. Consente di confrontare gli elementi candidati con il minimo attuale.
minIndex Memorizza la posizione del minimo trovato fino a quel momento. Non contiene il valore minimo, ma l'indice in cui quel valore si trova.
temp Variabile di appoggio per effettuare lo scambio. Evita la perdita del valore originariamente presente in numbers[i].

7. Complessità e prestazioni

Aspetto Valore Significato
Tempo nel caso migliore O(n²) Anche se il vettore è già ordinato, il minimo va comunque cercato ogni volta.
Tempo nel caso medio O(n²) Il numero di confronti resta quadratico.
Tempo nel caso peggiore O(n²) Su insiemi grandi diventa rapidamente poco efficiente.
Numero di confronti n(n-1)/2 I confronti effettuati sono sempre gli stessi, perché il minimo viene cercato completamente a ogni iterazione.
Numero massimo di scambi n-1 Rispetto ad altri algoritmi semplici, il numero di scambi è contenuto.
Spazio extra O(1) Usa solo poche variabili aggiuntive.
Stabilità No In generale non preserva l'ordine relativo di elementi uguali.
Adattività No Non trae vero vantaggio dal fatto che il vettore sia già quasi ordinato.
Il costo quadratico deriva dalla somma dei confronti eseguiti dal ciclo interno: (n-1) + (n-2) + ... + 1. Questa somma vale n(n-1)/2, quindi cresce proporzionalmente a n².
Il Selection Sort è utile per capire la logica dell'ordinamento, ma non è l'algoritmo da preferire quando i dati sono molti.

8. Confronto con altri algoritmi

Conviene confrontarlo subito con altri algoritmi semplici, così non viene interpretato come unico modello possibile.

Algoritmo Idea principale Punto forte Limite principale
Selection Sort Cerca il minimo e lo porta davanti Molto leggibile e facile da spiegare O(n²) anche quando i dati sono quasi ordinati
Bubble Sort Confronta elementi adiacenti e li fa “salire” o “scendere” Visivamente intuitivo Molti scambi inutili
Insertion Sort Inserisce ogni elemento nella posizione corretta della parte già ordinata Molto valido su insiemi piccoli o quasi ordinati Resta quadratico nel caso peggiore

Quando è utile studiarlo

Quando si vuole introdurre il concetto di minimo, indice, scambio e parte ordinata/non ordinata del vettore.

Quando non conviene usarlo davvero

Quando si lavora con molti dati o quando il linguaggio mette già a disposizione algoritmi migliori, come std::sort().

9. Errori tipici

Errore 1 • Confondere minimo e indice del minimo

Molti studenti salvano direttamente il valore minimo, ma poi non sanno più con quale posizione fare lo scambio.

Errore 2 • Far partire il ciclo interno da 0

Il ciclo interno deve partire da i + 1, non da 0, altrimenti si ricontrolla inutilmente anche la parte già sistemata.

Errore 3 • Scambiare ad ogni confronto

Nel Selection Sort lo scambio va fatto dopo aver trovato il minimo dell'intera parte non ordinata, non ogni volta che si incontra un valore più piccolo.

Errore 4 • Dimenticare che l'ultimo passaggio non richiede ricerca completa

Per questo il ciclo esterno arriva fino a n - 2.

12. Riepilogo

  • Il Selection Sort è un algoritmo di ordinamento per confronto.
  • Ad ogni passaggio cerca il minimo nella parte non ordinata.
  • Lo scambio viene fatto solo alla fine del ciclo interno.
  • Lavora in-place e usa pochissima memoria aggiuntiva.
  • Ha complessità temporale O(n²).
  • È molto utile per capire la logica degli ordinamenti, ma non è la soluzione più efficiente in pratica.
Quando il meccanismo del Selection Sort risulta davvero chiaro, si costruisce già una base importante per confrontare ordinamenti diversi e per capire meglio perché algoritmi più evoluti risultano preferibili in molti casi reali.
